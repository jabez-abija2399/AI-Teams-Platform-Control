# Development Rules


# Before Coding


Always:

1. Understand requirement
2. Review architecture
3. Create plan
4. Identify risks


# Coding Standards


Code must be:


- Clean
- Readable
- Modular
- Reusable
- Type safe


# Components


Rules:


One component:

One responsibility


Avoid:

Huge components

Repeated UI

Mixed business logic



# Functions


Functions should:

- Have clear purpose
- Be testable
- Be reusable



# File Rules


Use meaningful names.


Avoid:


utils2.ts

temp.ts

final.ts



# Error Handling


Always handle:

- Loading
- Errors
- Empty states



# Changes


Never rewrite working systems without reason.


Always explain architectural changes.

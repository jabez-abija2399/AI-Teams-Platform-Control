# 1. Identity

## Project Name

AI Teams Platform


## Vision

Create the world's most advanced AI-native software development company platform.

The platform allows individuals, startups, and organizations to describe software ideas and receive complete software products created by specialized AI teams.


## Mission

Transform software creation from:

Idea → Human Planning → Human Development → Human Testing → Human Deployment

into:

Idea → AI Company → AI Planning → AI Engineering → AI Verification → Production Software


# 2. Core Belief

AI should not behave as a simple coding assistant.

AI should operate as a complete software organization.

The platform must create the experience of working with:

- CEO
- Product Manager
- Software Architect
- UI/UX Designer
- Frontend Engineers
- Backend Engineers
- Database Engineers
- QA Engineers
- Security Engineers
- DevOps Engineers


# 3. Product Philosophy

The platform follows three principles:

## Intelligence

AI must understand problems, not only generate output.

## Organization

AI agents must collaborate through defined roles and workflows.

## Accountability

Every action must have:

- Owner
- Reason
- Output
- Verification


# 4. Engineering Principles


## Documentation First

Documentation is part of the product.

Every major:

- Architecture decision
- Feature
- Workflow
- Agent behavior

must be documented.


## Modular Architecture

The system must be designed as independent modules.

Initial approach:

Modular Monolith

Future:

Microservices


## Quality Over Speed

Fast development without quality creates technical debt.

Every feature requires:

- Validation
- Testing
- Documentation


# 5. AI Rules

Every AI agent must:

1. Read project documentation before working.

2. Understand existing systems.

3. Respect boundaries.

4. Avoid unnecessary complexity.

5. Update documentation.

6. Verify changes.


AI agents cannot:

- Rewrite architecture without approval.
- Delete systems without analysis.
- Create duplicate solutions.
- Skip testing.


# 6. Long-Term Vision


The final platform should become:

"An AI software company that can create software companies."


Future capabilities:

- Autonomous product teams
- AI project managers
- AI developers
- AI security teams
- AI operations teams
- Self-improving engineering workflows

# 8. Success Definition


The project succeeds when:

A user can describe a software idea and the AI organization can:

- Understand requirements
- Design architecture
- Create implementation plans
- Generate software
- Test software
- Deploy software
- Maintain software


# 9. Final Principle


Every contributor, human or AI, must think:

"I am building a long-term AI software company, not only writing code."
